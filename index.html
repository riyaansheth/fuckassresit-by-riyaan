Backtracking / Hard

N-Queens

Exam trigger: Place queens so none attack each other.

Plain-English Core

Place one queen per column. Before placing, check if its row or diagonals are already under attack. If a choice later fails, undo it and try the next row.

Memory lock: Choose, mark, recurse, unmark.

Dry run: For n=4, the recursion eventually finds two valid board layouts.

Code Walkthrough

  1. Column j is the recursion level.
  2. Try every row i in that column.
  3. Reject if row, main diagonal, or anti-diagonal is occupied.
  4. Place queen and mark row/diagonals.
  5. Recurse to next column, then unmark on return.

Roughly O(n!), space O(n) plus result storage.

Marks-Losing Traps

  • Diagonal formulas: row+col and row-col+n.
  • Backtracking bugs usually come from forgetting to unmark.

Pattern Matching / Classic

Rabin-Karp

Exam trigger: Find pattern occurrences in text.

Plain-English Core

Instead of comparing every window character by character, compute a fingerprint for the pattern and each text window. If fingerprints differ, skip. If they match, verify characters to avoid hash collision.

Memory lock: Hash first, verify only on hash match.

Dry run: For text ABABDABABCABAB and pattern ABABCABAB, rolling hash lands on the real match.

Code Walkthrough

  1. Compute hash of pattern and first text window.
  2. Slide the window one character at a time.
  3. If hashes match, compare actual characters.
  4. Update text hash by removing outgoing char and adding incoming char.
  5. Fix negative hash by adding modulus.

Average O(n+m), worst O(n*m) with many collisions. Space O(1).

Marks-Losing Traps

  • Always verify characters after hash match.
  • Handle negative modulo after rolling update.

Pseudocode — write this on paper

FUNCTION rabinKarp(text, pattern):
  M <- length(pattern); N <- length(text)
  d <- 256          // alphabet size
  q <- 101          // prime modulus
  h <- 1
  FOR i <- 1 TO M - 1:
    h <- (h * d) MOD q          // h = d^(M-1) MOD q
  p <- 0; t <- 0                // hash of pattern, hash of first window
  FOR i <- 0 TO M - 1:
    p <- (d * p + pattern[i]) MOD q
    t <- (d * t + text[i]) MOD q
  FOR i <- 0 TO N - M:
    IF p == t:                  // hashes match, verify char by char
      IF text[i .. i+M-1] == pattern:
        REPORT match at index i
    IF i < N - M:               // roll hash to next window
      t <- (d * (t - text[i] * h) + text[i+M]) MOD q
      IF t < 0:
        t <- t + q

UNIT 03

Dynamic Programming / Classic

Travelling Salesman Problem

Exam trigger: Visit every city exactly once and return to start with minimum cost.

Plain-English Core

The mask is a checklist of visited cities. From your current city, try every unvisited city and remember the cheapest result for this exact state.

Memory lock: TSP state is: where am I, and who have I visited?

Dry run: For 4 cities there are 4 * 2^4 states, much smaller than raw permutations.

Code Walkthrough

  1. State: dp[pos][mask] = minimum remaining cost from pos after visiting mask.
  2. Base: if all cities visited, return cost[pos][0].
  3. Try every unvisited city i.
  4. Answer is min(cost[pos][i] + solve(i, mask with i)).
  5. Memoize to avoid recomputing the same state.

Time O(n^2 * 2^n), space O(n * 2^n).

Marks-Losing Traps

  • Use bit operations carefully: mask | (1 << i).
  • This is exponential DP, suitable only for small n.

Pseudocode — write this on paper

FUNCTION tsp(cost):
  n <- number of cities
  dp <- 2D array [1<<n][n] filled with -1   // dp[mask][pos]
  RETURN solve(mask = 1, pos = 0)   // only city 0 visited

FUNCTION solve(mask, pos):
  // all cities visited: close the tour back to city 0
  IF mask == (1 << n) - 1:
    RETURN cost[pos][0]
  IF dp[mask][pos] != -1:
    RETURN dp[mask][pos]
  best <- INFINITY
  FOR next <- 0 TO n - 1:
    IF (mask AND (1 << next)) == 0:   // next not yet visited
      best <- MIN(best, cost[pos][next] + solve(mask OR (1 << next), next))
  dp[mask][pos] <- best
  RETURN best

    static int totalCost(int mask, int pos, int n, int[][] cost, int[][] dp) {

        // Base case: if all cities are visited, return the
        // cost to return to the starting city (0)
        if (mask == (1 << n) - 1) {
            return cost[pos][0];
        }

        if (dp[pos][mask] != -1) {
            return dp[pos][mask];
        }

        int ans = Integer.MAX_VALUE;

        // Try visiting every city that has not been visited
        // yet
        for (int i = 0; i < n; i++) {
            if ((mask & (1 << i)) == 0) {

                // If city i is not visited, visit it and
                // update the mask
                ans = Math.min(ans, cost[pos][i] +
                        totalCost(mask | (1 << i), i, n, cost, dp));
            }
        }

                String.format("%4s", Integer.toBinaryString(mask)).replace(' ', '0'),
                ans);
        dp[pos][mask] = ans;
        return ans;
    }

Find Minimum in Rotated Sorted Array

Exam trigger: Sorted array, rotated, unique values, asks minimum in O(log n).

Plain-English Core

Imagine a sorted shelf of books was cut once and the back half was moved to the front. The smallest book is exactly where the shelf breaks. Compare the middle book with the rightmost book: if middle is bigger, the break is to the right; otherwise the break is at middle or left.

Memory lock: Rotated minimum rule: MID bigger than RIGHT means GO RIGHT. Otherwise, keep MID.

Dry run: For [3,4,5,1,2], mid=5 and right=2, so jump right. Then [1,2], mid=1 <= 2, keep left. Answer 1.

Code Walkthrough

  1. Keep start and end around the possible minimum.
  2. Pick mid.
  3. If nums[mid] > nums[end], the left side is the big rotated chunk, so minimum is after mid.
  4. Else mid belongs to the small sorted chunk, so keep mid by moving end = mid.
  5. When start == end, that index is the minimum.

Time O(log n), space O(1).

Marks-Losing Traps

  • Use end = mid, not mid - 1, because mid may be the minimum.
  • This version assumes unique values. Duplicates need extra handling.

Pseudocode — write this on paper

FUNCTION findMin(array):
  n <- length(array)
  IF n == 1:
    RETURN array[0]
  start <- 0
  end <- n - 1
  WHILE start < end:
    mid <- start + (end - start) / 2
    IF array[mid] > array[end]:      // minimum lies to the right of mid
      start <- mid + 1
    ELSE:                            // mid could be the minimum, keep it
      end <- mid
  RETURN array[end]

    public static boolean search(int[] nums, int target) {
        if (nums == null || nums.length == 0){
            return false;
        }
        int start = 0, end = nums.length - 1;
        while (start <= end){
            int mid = start + (end - start) / 2;
            if (nums[mid] == target){
                return true;
            }
            if (nums[start] == nums[mid] && nums[mid] == nums[end]) {
                start++;
                end--;
            }
            else if (nums[mid] >= nums[start]){
                if (target >= nums[start] && target < nums[mid]){

                if (target > nums[mid] && target <= nums[end]){
                    start = mid + 1;
                }
                else{
                    end = mid - 1;
                }
            }
        }

Exam trigger: Jobs have deadline and profit, each takes one slot.

Plain-English Core

Treat time slots like parking spaces before deadlines. Process the most profitable jobs first and park each job as late as possible before its deadline, so earlier spaces remain available for tighter jobs.

Memory lock: Richest job first. Park it as late as possible.

  2. For each job, scan backward from min(deadline, maxDeadline) - 1.

  4. Add its profit when placed.

  • Do not place a job after its deadline.
  • Latest possible slot is better than earliest possible slot.

Dijkstra's Algorithm

Exam trigger: Single-source shortest path, all edge weights non-negative.

Plain-English Core

You expand the nearest known city first. Once the smallest distance comes out of the priority queue, all longer roads cannot beat it anymore, so you relax its neighbors.

Memory lock: Dijkstra is Google Maps with no negative roads.

Dry run: From source 0, every relaxation says: is dist[u] + weight(u,v) smaller than dist[v]?

Code Walkthrough

  1. Set all distances to infinity, source to 0.
  2. Push source into a min-priority queue.
  3. Pop the node with smallest known distance.
  4. For each neighbor, if going through current is cheaper, update distance and push it.
  5. Continue until the queue is empty.

With adjacency list and priority queue: O((V+E) log V), space O(V+E).

Marks-Losing Traps

  • Dijkstra is invalid with negative edges.
  • A production implementation usually skips stale queue entries.

Pseudocode — write this on paper

FUNCTION dijkstra(adj, V, src):
  dist[0 .. V-1] <- INFINITY
  dist[src] <- 0
  pq <- min-priority-queue ordered by distance
  pq.push((src, 0))
  WHILE pq is not empty:
    u <- pq.pop().node        // vertex with smallest distance
    FOR each edge (v, weight) IN adj[u]:
      IF dist[u] + weight < dist[v]:
        dist[v] <- dist[u] + weight
        pq.push((v, dist[v]))
  RETURN dist

Prim's Algorithm

Exam trigger: Connect all vertices with minimum total edge cost.

Plain-English Core

Start with one island. Always buy the cheapest bridge that connects your island to a new island. Never buy a bridge that connects two places already inside your island.

Memory lock: Prim grows one connected tree by cheapest outside edge.

Dry run: Each pop asks: does this edge bring a new vertex? If yes, pay for it.

  1. Start from any source with cost 0.
  2. Push outgoing edges into a min-priority queue.
  3. Pop cheapest edge.
  4. If destination is unvisited, add its weight to MST.
  5. Push edges from that newly visited node.

  • Prim solves MST, not shortest path.
  • Visited check prevents cycles.

Zero-One BFS Algorithm

Exam trigger: Graph shortest path where weights are only 0 and 1.

Plain-English Core

A zero-cost road should be explored immediately, like a free move. A one-cost road waits at the back. A deque gives you this behavior without a full priority queue.

Memory lock: 0 goes front, 1 goes back.

Dry run: If moving to v costs 0, v deserves the same urgency as current. If it costs 1, it waits.

Code Walkthrough

  1. Initialize distances with infinity and source with 0.
  2. Use a deque.
  3. When relaxing a 0-weight edge, push the neighbor to the front.
  4. When relaxing a 1-weight edge, push the neighbor to the back.

Time O(V+E), space O(V+E).

Marks-Losing Traps

  • Only use this for weights 0 and 1.
  • For general non-negative weights, use Dijkstra.

import java.util.ArrayList;
import java.util.Deque;
import java.util.List;

public class ZeroOneBFS {
    private static class Vertex {
        int dest;
        int weight;

        public Vertex(int dest, int weight) {
            this.dest = dest;
            this.weight = weight;
        }
    }

    private static final int numVertex = 9;
    private List<Vertex>[] edges = new ArrayList[numVertex];

    public ZeroOneBFS() {
        for (int i = 0; i < edges.length; i++) {
            edges[i] = new ArrayList<Vertex>();
        }
    }

    public void addEdge(int u, int v, int wt) {
        edges[u].add(edges[u].size(), new Vertex(v, wt));
        edges[v].add(edges[v].size(), new Vertex(u, wt));
    }

    public void zeroOneBFS(int src) {
        int[] dist = new int[numVertex];
        for (int i = 0; i < numVertex; i++) {
            dist[i] = Integer.MAX_VALUE;
        }

        Deque<Integer> queue = new ArrayDeque<Integer>();
        dist[src] = 0;
        queue.addLast(src);

        while (!queue.isEmpty()) {
            int v = queue.removeFirst();
            for (int i = 0; i < edges[v].size(); i++) {

                if (dist[edges[v].get(i).dest] >
                        dist[v] + edges[v].get(i).weight) {

                    dist[edges[v].get(i).dest] =
                            dist[v] + edges[v].get(i).weight;

                    if (edges[v].get(i).weight == 0) {
                        queue.addFirst(edges[v].get(i).dest);
                    } else {
                        queue.addLast(edges[v].get(i).dest);
                    }
                }
            }
        }

        for (int i = 0; i < dist.length; i++) {
            System.out.print(dist[i] + " ");
        }
    }

Coin Change

Exam trigger: Unlimited coins, minimum number to make amount.

Plain-English Core

Build every amount from 0 upward. If you know the best way to make amount i - coin, then amount i can be made by adding one more coin.

Memory lock: To pay i, ask: what if my last coin was c?

Dry run: For 11 with [1,2,5], dp[11] becomes dp[6]+1, then dp[9]+1, then dp[10]+1 = 3.

Code Walkthrough

  1. dp[x] = minimum coins needed for amount x.
  2. Set dp[0] = 0 and others to amount + 1 as fake infinity.
  3. For every amount i, try every coin c.
  4. If i >= c, dp[i] = min(dp[i], dp[i-c] + 1).
  5. If dp[amount] is still fake infinity, return -1.

Time O(amount * numberOfCoins), space O(amount).

Marks-Losing Traps

  • This is minimum count, not number of ways.
  • Initialize with amount + 1, not Integer.MAX_VALUE, to avoid overflow on +1.

Plain-English Core

Exam trigger: Two strings, longest subsequence common to both.

Plain-English Core

Compare prefixes of both strings. If the last characters match, they can be part of the answer. If not, drop one last character from either string and keep the better result.

Plain-English Core

For every number, look left and ask: which smaller number can I stand behind? The best chain ending here is one more than the best chain ending at that smaller number.

Plain-English Core

Each position tracks two facts: the best chain length ending here, and how many ways can produce that length. A better previous chain replaces the count; an equally good chain adds to the count.

Bellman-Ford

Exam trigger: Single-source shortest path with negative edges or negative cycle detection.

Plain-English Core

Shortest paths with at most 1 edge, then at most 2 edges, then at most 3 edges. After V-1 rounds, any normal shortest path is finished. If it still improves on round V, a negative cycle exists.

Memory lock: Bellman-Ford is edge relaxation by patience: V-1 rounds, then cycle check.

Dry run: Every round lets paths use one more edge. No shortest simple path needs more than V-1 edges.

Code Walkthrough

  1. Initialize dist[source] = 0 and all others infinity.
  2. Repeat V-1 times: try relaxing every edge.
  3. Relax means if dist[u] + w < dist[v], update dist[v].
  4. Do one more pass; if any distance improves, report negative cycle.

Time O(V*E), space O(V).

Marks-Losing Traps

  • Check dist[source] != infinity before adding weight.
  • Dijkstra cannot handle negative edges; Bellman-Ford can.

Pseudocode — write this on paper

FUNCTION bellmanFord(V, edges, src):
  dist <- array of size V filled with INFINITY
  dist[src] <- 0
  // Relax all edges V-1 times
  FOR i <- 1 TO V - 1:
    FOR each (u, v, w) IN edges:
      IF dist[u] != INFINITY AND dist[u] + w < dist[v]:
        dist[v] <- dist[u] + w
  // One more pass: any relaxation means a negative cycle
  FOR each (u, v, w) IN edges:
    IF dist[u] != INFINITY AND dist[u] + w < dist[v]:
      RETURN "negative cycle detected"
  RETURN dist

Exam trigger: Shortest path between every pair of vertices.

Plain-English Core

For every possible middle city k, ask every pair i to j: is going i -> k -> j cheaper than the route we already know?

Memory lock: Can city k be a useful shortcut?

Dry run: When k=2, every pair gets permission to route through vertex 2.

  1. dist starts as the adjacency matrix.
  2. For each intermediate k, loop all i and j.
  3. If dist[i][k] + dist[k][j] < dist[i][j], update it.

0/1 Knapsack DP

Exam trigger: Each item can be taken once, maximize value under capacity.

Plain-English Core

For each item, decide whether the bag improves if you include it. Loop capacity backward so the same item cannot be reused in the same round.

Memory lock: 0/1 means backward capacity loop. Unlimited means forward loop.

Dry run: Capacity 4, item weight 3 value 8 can improve dp[3] and dp[4], but backward loop prevents double-use.

Code Walkthrough

  1. dp[c] = best value for capacity c.
  2. For each item, loop c from capacity down to itemWeight.
  3. dp[c] = max(dp[c], dp[c-weight] + value).
  4. Answer is dp[capacity].

Time O(n*capacity), space O(capacity).

Marks-Losing Traps

  • Backward loop is the key 0/1 detail.
  • Do not confuse with fractional knapsack; ratio greedy is wrong here.

Pseudocode — write this on paper

FUNCTION knapsack(capacity, values, weights):
  n <- length(weights)
  dp <- array of size (capacity + 1) filled with 0
  // dp[c] = best value using items seen so far within capacity c
  FOR i <- 0 TO n - 1:
    // iterate capacity downward so each item is used at most once
    FOR c <- capacity DOWNTO weights[i]:
      dp[c] <- MAX(dp[c], dp[c - weights[i]] + values[i])
  RETURN dp[capacity]

Subset Sum

Exam trigger: Can some subset make target sum? Sometimes print one subset.

Plain-English Core

For every prefix of numbers and every target sum, record whether that sum is possible. Then walk backward through the table to find which numbers were used.

Memory lock: For each number: ignore it or use it.

Dry run: Target 12 with [1,3,17,4,5,18,20] can use 3+4+5.

Code Walkthrough

  1. dp[i][j] = can first i numbers make sum j?
  2. dp[i][0] = true because empty subset makes 0.
  3. If current number is too large, copy dp[i-1][j].
  4. Else dp[i][j] = dp[i-1][j] OR dp[i-1][j-current].
  5. To print subset, walk backward: if dp[i-1][j] is false, current was used.

Time O(n*sum), space O(n*sum).

Marks-Losing Traps

  • Works well when target sum is not huge.
  • Printing one subset requires the full table or parent tracking.

Pseudocode — write this on paper

FUNCTION subsetSum(nums, sum):
  n <- length(nums)
  dp <- 2D boolean array [n+1][sum+1]
  // empty subset always reaches sum 0
  FOR i <- 0 TO n:
    dp[i][0] <- true
  FOR i <- 1 TO n:
    FOR j <- 1 TO sum:
      IF j < nums[i-1]:
        dp[i][j] <- dp[i-1][j]              // cannot take item i
      ELSE:
        dp[i][j] <- dp[i-1][j] OR dp[i-1][j - nums[i-1]]  // skip OR take
  RETURN dp[n][sum]
